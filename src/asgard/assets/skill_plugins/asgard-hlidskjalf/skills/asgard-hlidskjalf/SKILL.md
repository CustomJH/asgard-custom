# asgard-hlidskjalf — 👁 시스템 아키텍처 검증

흘리드스캴프는 전 세계를 조망하는 자리다 — 파일 하나가 아니라 의존 그래프 전체를 본다.

이 팩은 **검증 절차 정본**이다. 빌스키르니르가 "구조를 어느 쪽으로 고정할지"의 정책이라면, 이 팩은 "현 구조가 선언된 아키텍처와 일치하는지"를 판정하는 절차다. 인상 비평은 산출물이 아니다 — 모든 발견은 파일:라인 + 재현 명령을 동반해야 한다.

## 검증 계약 (전 방 공통)

1. **선언 먼저** — 검증 대상의 선언된 아키텍처를 찾는다: ARCHITECTURE/README/ADR·계층 테스트·모듈 규칙. 선언이 없으면 관측으로 사실상의 계층을 복원해 명기하고, 그 복원을 판정 기준으로 선언한 뒤 시작한다(기준 없는 위반 판정은 무효).
2. **증거 우선** — 발견 1건 = `〈유형〉 파일:라인 — 근거 명령 + 출력 요지`. 재현 명령 없는 발견은 보고에 싣지 않는다.
3. **심각도 3단** — `구조 결함`(역류·순환·경계 우회 — 방치 시 전파) / `부채`(god module·산탄 수정 신호 — 계획 수리) / `관찰`(단독 무해, 추세 기록).
4. **봉인 제안** — 확정된 위반은 사람이 다시 잡지 않도록 fitness function(계층 테스트·lint 규칙·CI 검사)으로 봉인하는 제안 1줄을 붙인다.
5. **판정 불능 = 미판정** — 그래프 추출이 실패한 영역은 PASS 로 뭉개지 않고 "미검증 영역"으로 명기한다 (fail-closed).
