# BOUNDARIES — 모듈 경계·계약·데이터 흐름 검증

경계의 불변식: **모듈은 공개 표면으로만 말한다**. 경계 우회 한 건이 캡슐화 전체를 무효화한다.

## 절차

1. **공개 표면 확정** — 모듈별 의도된 공개 표면을 찾는다: `__init__.py`/`__all__`, `index.ts` 배럴, `exports`(package.json), JVM 공개 패키지. 선언이 없으면 "타 모듈이 실제로 참조하는 심볼 집합"을 관측 표면으로 복원해 명기한다.
2. **우회 참조 검출** — 공개 표면 밖 내부 심볼 직접 참조를 찾는다: `grep -rn "from .*\.internal\|from .*\._"` (Python 밑줄 모듈), deep import(`import x from "pkg/src/…"`), 타 모듈 private 클래스 참조. 각 건 = `우회 파일:라인 — 〈내부 심볼〉`.
3. **데이터 소유권** — 저장소(테이블·컬렉션·파일)마다 쓰기 주인이 하나인지 대조한다: 같은 테이블에 두 모듈이 직접 쓰면 경계 붕괴다. ORM 모델·스키마 참조를 grep 으로 전수 추적한다.
4. **흐름 방향** — 선언된 단방향 흐름(예: FE→BE→외부, UI→서비스→저장)의 역방향 호출을 찾는다. 콜백·이벤트로 위장한 역방향 동기 의존(하위가 상위 완료를 기다림)도 동일 판정.
5. **계약 드리프트** — 경계를 넘는 계약(API 스키마·이벤트 페이로드·DTO)이 소비자와 어긋나는지: 스키마 정의와 실사용 필드를 대조하고, 도메인 객체가 DTO 없이 경계를 그대로 넘는 누수(내부 표현이 외부 계약이 되는 것)를 기록한다.

## 판정 기준

- 우회 참조·이중 쓰기 주인·역방향 흐름 = `구조 결함` — 전파 전 수리 대상.
- 도메인→DTO 누수·과대 공개 표면(참조 0 인 export 다수) = `부채`.
- 관측 표면으로만 복원된 경계(선언 부재) = `관찰` + 표면 선언(배럴·`__all__`) 봉인 제안.

## 봉인

- 우회 참조: lint 규칙(`no-restricted-imports`, import-linter 계약, ArchUnit `noClasses().should().accessClassesThat()`)으로 경로 패턴을 금지.
- 이중 쓰기: 소유 모듈 경유 API 로 강제하고, 스키마 참조 grep 을 CI 검사로 고정.
- 계약: 경계 스키마를 스냅샷 테스트(직렬화 골든 파일)로 고정해 무단 드리프트를 잡는다.
