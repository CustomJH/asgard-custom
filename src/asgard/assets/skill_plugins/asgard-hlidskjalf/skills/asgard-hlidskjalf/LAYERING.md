# LAYERING — 계층·의존 방향·순환 검증

계층 아키텍처의 불변식은 하나다: **의존은 한 방향으로만 흐른다**. 검증은 그 불변식의 반례 탐색이다.

## 절차

1. **계층 배정표 확정** — 선언(문서·계층 테스트)이 있으면 그대로, 없으면 최상위 모듈/패키지를 전수 나열하고 역할로 계층을 복원한다. **미배정 모듈 0** 이 첫 판정 항목이다 — 어느 계층에도 속하지 않는 모듈은 그 자체가 발견이다.
2. **import 그래프 추출** — 생태계별 정본 경로:
   - Python: `python -c "import ast"` 기반 스크립트로 모듈별 top-level import 를 뽑는다(문자열 grep 는 조건부/지연 import 를 오판). 빠른 1차는 `grep -rn "^from \|^import " --include="*.py"`.
   - JS/TS: `npx madge --circular src/` (순환), `npx madge --json src/` (그래프). tsconfig paths 별칭을 madge 설정에 반영해야 오탐이 없다.
   - JVM: `jdeps -verbose:class` 또는 ArchUnit 테스트. Go: `go list -deps`, `import cycle` 은 컴파일러가 이미 봉인.
3. **역류 검출** — 그래프의 각 간선 (A→B) 에 계층 순위를 입혀 `rank(A) < rank(B)` (하위가 상위를 참조) 간선을 전부 나열한다. 각 건은 `역류 파일:라인 — A(계층) → B(계층)` 로 기록.
4. **순환 검출** — 계층 내부 순환도 결함이다(모듈 단위). 순환 고리 전체를 나열하고 최소 절단 간선(어느 참조를 인터페이스 역전으로 끊을지)을 제안한다.
5. **탈출구 감사** — 함수 내부 지연 import·동적 로드는 허용된 탈출구인지(선언 존재) 무단 우회인지 구분해 기록한다. 무단이면 `부채`.

## 판정 기준

- 역류 간선 1건 이상 = `구조 결함`. "동작하니까"는 반론이 아니다 — 역류는 컴파일이 아니라 변경 파급에서 터진다.
- 순환 고리 = `구조 결함`(2모듈 상호참조 포함). 지연 import 로 숨긴 순환도 동일 판정.
- 미배정 모듈 = `부채`. 계층 테스트 부재 = `관찰` + 봉인 제안 필수.

## 봉인 (fitness function)

확정 위반은 테스트로 봉인한다 — 계층 배정표(frozenset)와 AST 기반 상향 import 검사를 가진 `test_architecture` 류 테스트가 정본 패턴이다: ① 전 최상위 모듈이 계층에 배정됨 ② 상향 top-level import 0건 ③ 허용 탈출구(지연 import)만 예외. JS/TS 는 `madge --circular` 를 CI 게이트로, JVM 은 ArchUnit 규칙으로 같은 불변식을 고정한다.
