# Vanadis Restraint

Treat simplicity as the product feature. Start from the simplest working reference, not from the busier revision.

## Constraints

1. Preserve the reference's section count, reading order, CTA count, and approximate page height.
2. Add no cards, dashboards, counters, status boards, decorative containers, or new full-width sections.
3. Spend at most two identity moves across the whole page:
   - one typographic or color signature;
   - one short evidence cue integrated into existing content.
4. Keep body copy, controls, and layout neutral enough that the signature remains visible.
5. Keep motion tactile and local: CTA feedback and one restrained entrance sequence are enough.
6. Reuse native HTML and CSS. Add no dependency, image, icon, SVG, canvas, or gradient unless the reference already needs it.

## Workflow

1. Capture the simple reference at desktop and mobile sizes.
2. List what already works in one sentence. Do not redesign those parts.
3. Choose the smallest identity gap that prevents the page from feeling like Freyja.
4. Make the two allowed identity moves inside the existing structure.
5. Compare reference and result side by side at the same viewport.
6. Remove an addition if it creates a new reading stop or makes the result more than 10% taller.
7. Verify the primary CTA, keyboard focus, mobile flow, and reduced-motion behavior.

## Acceptance check

- A viewer should recognize the reference layout before noticing the refinements.
- The page should explain Freyja without a separate evidence module.
- Removing either identity move should make the page less distinctive; otherwise remove it.
- No P0, P1, or P2 visual or interaction defect may remain.
