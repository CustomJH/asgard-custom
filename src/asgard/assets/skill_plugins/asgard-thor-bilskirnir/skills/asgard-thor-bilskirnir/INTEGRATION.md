# 외부 연동 경계 정책

## 아키텍처 흐름 (MANDATORY)

```
{project}-fe (프론트엔드) → /api/v1/* (백엔드) → 외부 서비스
```

프론트엔드는 백엔드 API 만 호출한다. 외부 서비스 직행 금지.

## 금지 패턴표

| 금지 | 대신 | 이유 |
|---|---|---|
| 프론트엔드 → 외부 API 직접 호출 | 프론트엔드 → 백엔드 → 외부 | API 키는 서버에만 존재해야 한다 |
| 에이전트 도구·임시 브리지를 연동 경로로 사용 | 백엔드 REST 엔드포인트 | 재현 가능·테스트 가능·배포 가능해야 한다 |
| 하드코딩 키·시크릿 | 환경변수 `${…}` | 보안·환경별 구성 |
| 프론트엔드 코드에 시크릿 | `application-{profile}.yml` 만 | 프론트엔드는 브라우저로 배포된다 |
| 서비스 추측 후 코딩 | 코딩 전 서비스·API 확정 | 비싼 방향 전환 방지 |

## 연동 결정 게이트 — 코드 작성 전 STOP

1. **특정** — 정확한 서비스를 명시한다 ("어떤 AI 서비스"가 아니라 "OpenAI Chat via 공식 SDK").
2. **검색** — 기존 연동이 이미 있는지 코드베이스를 먼저 찾는다.
3. **계층 확인** — 외부 호출은 service 계층 소관이다.
4. **크리덴셜 확인** — 서버측 환경변수 참조만.
5. **프론트 접근 확인** — 프론트엔드는 `/api/v1/*` 경유만.

## 신규 외부 서비스 추가 절차

1. `.env.example` 에 환경변수 키 추가 (실값 금지)
2. service 계층 `config/` 에 타입 세이프 설정 바인딩 추가
3. domain `spec/` 에 서비스 인터페이스 추가
4. domain `dto/` 에 요청·응답 DTO 추가
5. service `logic/` 에 구현 추가
6. service `rest/` 에 REST 엔드포인트 추가
7. 프론트엔드 서비스 결선은 프론트 담당 표면 (백엔드 완료 후)
8. **NEVER** 크리덴셜을 프론트엔드 코드에 두지 않는다

호출 신뢰성(타임아웃·재시도·서킷 브레이커·응답 스키마 검증)은 `asgard-thor-lightning` 의 외부 연동 캐논을 합성 로드해 따른다.
