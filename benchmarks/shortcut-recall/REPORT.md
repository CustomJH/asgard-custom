# 숏컷 벤치 — recall 주입 on/off A/B (26-07-16, 36런)

## 질문
개인 메모리(Tier0) recall 주입이 읽기 과업의 **탐색을 실제로 단축하는가** — 정확도 손실 없이.

## 방법 (harness.py)
- 합성 저장소 49파일(디코이 어휘 포함) + 심은 사실 6종. **메모리는 위치만 기록, 값은 미기록**
  — 주입 arm 도 파일을 열어야(증거) 답할 수 있고, 절약되는 것은 탐색뿐이다.
- 6과업 × {주입 on, off} × 3반복 = **36런** (`asgard run --provider claude-native --json`, DIRECT).
- 런 간 `.asgard` 상태 초기화(route-priors 학습 전이 차단). 주입 여부는 usage 카운터 증분으로 검증.
- 0-LLM 사전검증: 과업 문구 6종 recall → 포인터 페이지 6/6 HIT.

## 결과 (results-36runs.jsonl — 런당 1행 append, 원본 그대로)
| 지표 | 주입 on | 주입 off | 델타 |
|---|---|---|---|
| 성공률 (교정 판정) | 18/18 | 18/18 | 동일 |
| 토큰 median | 28,335 | 85,562 | **−67%** |
| 벽시계 median | 12.4s | 39.8s | **−69%** |
| 파일 경로 인용(증거 보존) | 18/18 | 17/18 | — |

on-arm 토큰 분산 극소(28.0k~28.4k) = 탐색 소거의 직접 증거. off-arm 은 66k~116k 로 출렁임.

## 판정기 주의 (jsonl 의 `success` 필드를 그대로 믿지 말 것)
기록 시점 판정 regex 가 `\b1747\b` 형태였는데 한글이 뒤따르면(`1747초`) `\b` 가 경계를 못
잡는다 — cache-ttl/on/r0 1건이 **거짓 실패**로 기록돼 있다. 교정 판정은 숫자 값에
`(?<!\d)…(?!\d)` 를 쓴다 (tests/test_bench_fixtures.py 가 이 교정 판정으로 18/18 을 회귀 고정).

## 한계
- 합성 저장소 + 위치-포인터형 메모리 + 읽기 전용 Q&A 라는 유리한 조건의 측정이다.
- 이득의 상한은 "탐색 비용" — 탐색이 원래 짧은 과업의 이득은 작다.
- claude-native(구독) 단일 provider 실측. 라이브 재실행은 `python3 harness.py build/pilot/full`.
- `--files 36` 류 다른 벤치의 36 과 무관 — 이 36 은 런 수다.
